# 自動売買BOTプロジェクト ルール

## コーディング規約
- 言語: Python 3.10以上
- フォーマッター/リンター: `black`, `flake8` を使用
- 状態管理: 取引履歴や保有ポジション、最新の約定価格などはSQLiteに記録し、再起動時にも状態を復元できるようにすること

## 分析とコンテキストの管理
- 分析によって得られた知見やAPIの仕様、新しいルールは `docs/knowledges/` フォルダ内のMarkdownファイルに記録し、次回以降のセッションで優先的に参照すること
- コードの修正は最小差分で行い、必ず `tests/` 内の関連するユニットテストを実行・修正して動作を担保すること

## コマンド実行の許可
- 依存関係のインストール（`pip install`）やテストの実行（`pytest`）は許可する

## プロジェクト構造
- `config/` - 戦略パラメータ (YAML)
- `src/bitbank_bot/` - メインパッケージ
- `tests/` - pytestテスト
- `scripts/` - データ取得等のスクリプト
- `data/` - SQLiteデータベース、ログ
- `docs/knowledges/` - 分析知見ログ

## 重要な設計判断
- 取引所API: ccxt（enableRateLimit対応）
- 指標計算: pandas_ta
- Maker注文優先（-0.02%リベート）
- Bitbank APIは同一キーで直列処理されるため、逐次的な設計
